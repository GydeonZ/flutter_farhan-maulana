# Basic Version and Branch Management (Git)

1. lebih baik kita pull dlu sebelum membuat atau mengupdate fitur baru pada branch master supaya meminimalisir terjadinya conflict

2. conflict terjadi karena ada perubahan pada file yang sama dan baris yang sama

3. Kalau terjadi conflict kita memiliki 3 pilihan current change yaitu menggunakan kode kita, incoming change yaitu menggunakan kode yang di push oleh orang lain, dan merge change yaitu menggabungkan kedua kode tersebut.
